# AFO Mobile Visual Runtime v0.02

## Purpose

AFO Mobile Visual Runtime is a mobile-first frontend runtime for agent-built apps. It turns JSON scene specifications into touch-first Three.js web apps hosted on Cloudflare Workers.

## Live Runtime

https://afo-mobile-visual-runtime.jaredtechfit.workers.dev

## What v0.02 Adds

v0.02 adds the first discovery and connector layer over the v0.01 visual demo.

New routes:

- `/llms.txt`
- `/openapi.json`
- `/agent-actions`
- `/mcp/tools`
- `/mcp/schema`
- `/mcp/call`
- `/api/scene/validate`
- `/api/scene/optimize`
- `/api/compile`
- `/api/receipt`

## Tool Contract

The runtime exposes these tool-style actions:

- `create_mobile_scene_spec`
- `validate_scene_spec`
- `optimize_for_phone`
- `compile_three_mobile_page`
- `bind_d1_rows_to_objects`
- `bind_r2_images_to_textures`
- `generate_visual_receipt`

## Runtime Flow

1. Agent reads `/llms.txt`.
2. Agent reads `/mcp/tools` and `/mcp/schema`.
3. Agent submits a scene spec to `/api/scene/validate`.
4. Agent optimizes the spec with `/api/scene/optimize`.
5. Agent compiles the spec through `/api/compile` or `/mcp/call`.
6. Runtime returns generated HTML and a visual receipt.

## Role in AFO Stack

Dogfoodma is the frontend visual runtime seed for the AFO app factory.

It sits after the backend infrastructure tools:

`Sandbox Builder -> GitHub MCP -> Cloudflare MCP -> D1/KV/R2 -> Dogfoodma -> deploy receipt`
