# Dogfoodma v0.04 — MCP Manifest, Adapters, Smoke Receipts

## Purpose

v0.04 keeps the manually tested v0.03.1 cockpit baseline and adds a stronger MCP-style connector surface plus data adapter contracts.

## Added MCP Routes

- `/mcp/manifest`
- `/mcp/resources`
- `/mcp/prompts`
- `/mcp/tool-schema`

## Added Adapter Routes

- `/api/adapters/d1/preview`
- `/api/adapters/r2/manifest`
- `/api/adapters/kv/theme`
- `/api/scene/from-d1`
- `/api/assets/from-r2`

## Added Smoke / Receipt Routes

- `/api/smoke`
- `/api/smoke/browser-contract`
- `/api/receipt/visual`

## Design Constraint

v0.04 is additive. It should not regress the v0.03.1 browser cockpit.

## Acceptance Checks

- The cockpit still renders and manual controls still work.
- `/mcp/manifest` returns a connector-style manifest.
- `/mcp/resources` returns discoverable resources.
- `/mcp/prompts` returns prompt entries.
- `/mcp/tool-schema` returns tool schemas.
- `/api/smoke` returns a structured visual/runtime receipt.
- D1, R2, and KV adapter routes return deterministic preview payloads.
