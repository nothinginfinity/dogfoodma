# Dogfoodma v0.03.1 — Cockpit Restoration Patch

## Why this patch exists

v0.03 improved the schema/API layer but regressed the browser experience. The cockpit restoration patch restores the interactive frontend loop while preserving the MCP-style runtime routes.

## Restored UX

- real Three.js canvas
- mobile drag/fly controls
- Brake button
- Boost button
- Reset button
- speed label updates
- crosshair target detection
- tap/select node drawer
- working top nav panels
- Studio, Tools, Schema, Sample, Receipt panels

## Preserved Runtime Layer

- `/llms.txt`
- `/openapi.json`
- `/agent-actions`
- `/mcp/tools`
- `/mcp/schema`
- `/mcp/call`
- `/api/design/presets`
- `/api/design/compile`
- `/api/scene/export/three-json`
- `/api/scene/export/r3f`
- `/api/scene/export/gltf-manifest`
- `/api/compile`
- `/api/receipt`

## Acceptance Criteria

The browser must show an interactive cockpit, not a static shell. Navigation and cockpit controls must visibly work before a version is considered done.
